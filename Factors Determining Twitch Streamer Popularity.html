Introduction and Motivation¶

Gaming is one of the biggest industries in the world right now. Many like myself enjoy playing them, but many would also agree that watching others play games can be equally as entertaining. Platforms such as Twitch allow streamers to play games live for their audiences, allowing for further interaction between creators and their fans. Not only do audiences get to see their favorite games be played, but they also get to engage with their favorite internet personalities as well. Because of this, our goal is to find out what makes a large streamer garner the attention that they get. This is why the data we are working with will be from the top 1000 streamers of 2020: we will be using the data science pipeline to find and analyze any relationships that may be present between the popularity of a streamer and other factors regarding their stream and viewership.

The columns for our dataset now read as follows:

Step 3: Exploratory Data Analysis¶

First, we need to consider what we want to define as "popularity." Twitch ranks its streamers through a combination of concurrent viewers, followers, views, and time streamed. However, we will be going with the same ranking of popularity that the dataframe uses in our exploratory data analysis phase. Twitch's measurement of "popularity" consists of too many variables to work with in our analysis, so for the sake of simplicity, we will only be using the total watch time in minutes to determine popularity. This way, we have one specific variable we are looking for if we want better insight into what makes a streamer successful. Going off my intuition of how the platform works, I believe that the more content a streamer produces, the more likely they're going to garner views. So first, let's analyze the relationship between total watch time and stream time using Seaborn to create a scatterplot:

Before we get into machine learning, we must also acknowledge that correlation doesn't mean causation. For example, although the number of followers and watch time a streamer have a linear relationship with each other, this does not prove that one causes the other or vice versa.

Step 4: Machine Learning¶

For this step, we're going to use the variables we chose from our exploratory data analysis (Peak Viewers, Average Viewers, Followers, Followers Gained, Views Gained) to predict the watch time a Twitch streamer could get if they had a random set of values for the aforementioned variables.

For this part, we're going to want to use Scikit Learn to create a model for the data. In this specific example, we want to create a multiple regression model, since we have multiple different variables that are treating as independent variables. In order to do so, we split our dataframe into randomly created subsets that will be used for either training or testing:

In [137]:
train, test = train_test_split(df, train_size = 0.2)

We can now create a linear model from the training data and see how accurate it is in predicting watch time:

In [138]:
x_train = train[["Peak viewers", "Average viewers", "Followers", "Followers gained", "Views gained"]]
x_test = test[["Peak viewers", "Average viewers", "Followers", "Followers gained", "Views gained"]]
y_train = train["Watch time(Minutes)"]
y_test = test["Watch time(Minutes)"]
#this will create our linear regression model
reg = linear_model.LinearRegression()
reg.fit(x_train, y_train)
#now we will make predictions using our model on the randomly selected test data
y_train_pred = reg.predict(x_train)
y_test_pred = reg.predict(x_test)
