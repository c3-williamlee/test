Introduction and Motivation¶

Gaming is one of the biggest industries in the world right now. Many enjoy playing them, but many others like myself would also agree that watching others play games can be equally as entertaining. Platforms such as Twitch allow streamers to play games live for their audiences, allowing for further interaction between creators and their fans. Not only do audiences get to see their favorite games be played, but they also get to engage with their favorite internet personalities as well. Because of this, our goal is to find out how large streamers keep their fans watching their content. This is why the data we are working with will be from the top 1000 streamers of 2020: we will be using the data science pipeline to find and analyze any relationships that may be present between the popularity of a streamer and other factors regarding their stream and viewership.

Step 4: Machine Learning¶

For this step, we're going to use the variables we chose from our exploratory data analysis (Followers) to predict the watch time a Twitch streamer could get if they had a random set of values for the aforementioned variable.

For this part, we're going to want to use Scikit Learn to create a model for the data. In this specific example, we want to create a multiple regression model, since we have multiple different variables that we are treating as independent variables. In order to do so, we split our dataframe into randomly created subsets that will be used for either training or testing:

In [12]:
train, test = train_test_split(df, train_size = 0.2)

We can now create a linear model from the training data and see how accurate it is in predicting watch time. For the x training and testing set, the variables will be the same as the ones we picked out from before. The y training and testing set will use watch time, the dependent variable. We will then fit them into a linear regression model, and allow it to make a prediction given our samples:

In [53]:
#we assign each training set to their respective columns in the dataframe
x_train = train[["Followers"]]
x_test = test[["Followers"]]
y_train = train["Watch time(Minutes)"]
y_test = test["Watch time(Minutes)"]
#this will create our linear regression model
reg = linear_model.LinearRegression()
reg.fit(x_train, y_train)
#now we will make predictions using our model on the randomly selected test data
y_train_pred = reg.predict(x_train)
y_test_pred = reg.predict(x_test)

After getting our predictions for both the training and test sets, it's time to get some regression metrics from the model so that we can gauge how accurate it was in determining watch time. The metrics we'll be using are listed here on the Scikit Learn documentation: https://scikit-learn.org/stable/modules/model_evaluation.html#r2-score

We will only be calculating:

Explained Variance: the proportion to which our model accounts for the variation of a data set- the closer to 1, the better.

R-squared: how close the variables are to the regression model- again, the closer the value is to 1 the better.

Let's see what these evaluate to using metrics from Scikit Learn:

In [54]:
explained_variance=metrics.explained_variance_score(y_train, y_train_pred)
print('Explained Variance: ', explained_variance)    
print('r squared: ', reg.score(x_train, y_train))

Explained Variance:  0.4586447305557694
r squared:  0.4586447305557694

So far, the results from the training data doesn't seem to be that strong, as there is a lot of variance among the errors. Let's see how the test data does:

In [55]:
explained_variance=metrics.explained_variance_score(y_test, y_test_pred)
print('Explained Variance: ', explained_variance)    
print('r squared: ', reg.score(x_test, y_test))

Explained Variance:  0.35037717629012466

As mentioned before, we want both values to be as close to 1 as possible. However, with values as low as these, this is a clear indication that we are in need of a more refined model for calculating linear relationships, and that just going by the number of followers a streamer has is not enough to determine their total watch time.

In our exploratory data analysis and in the heat map, we've observed that there are multiple different independent variables that also have a strong correlation with watch time besides just Followers. These include Peak viewers, Average viewers, Followers gained, and Views gained. So it seems that instead of using just a simple linear regression model for our data, we need to use a multiple regression model. That way, we are able to explain the relationship between our single dependent variable and various other independent variables. Though it will be more difficult to visualize, it will hopefully allow us to create a better predictor for our values. So let's try again, this time by training x on multiple variables:

Now we need to test whether or not we should reject our null hypothesis, which states that there does not exist a relationship between peak viewers, average viewers, followers, followers gained, and views gained. We can do this by analyzing the relationship between the residuals for our new model (the distance between the actual plotted points and the linear regression model) and the total watch time in a histogram, which will show us the distribution of the differences between each point and the linear model:

In [68]:
residuals = (y_train - y_train_pred)
sbn.displot(residuals)
Out[68]:

Here, we see a mostly symmetrical distribution where most of the values are clustered around 0, which is how we expect the relationship between the residuals and the watch time should look like. Therefore, we can reject the null hypothesis, because this indicates a linear relationship between the set of independent variables and the dependent variable.

Step 5: Making Conclusions¶

Despite the name of this step, we cannot make any certain conclusions based on the patterns we've seen in the data. We have proven that there does indeed exist a linear relationship between a streamer's followers, average viewers, peak viewers, followers gained, and viewers gained, though there is a lot we can still unpack when questioning the certainty of this statement.

For example, we must also acknowledge that correlation doesn't mean causation. For example, although the number of followers and watch time a streamer have a linear relationship with each other, this does not prove that one causes the other or vice versa.
